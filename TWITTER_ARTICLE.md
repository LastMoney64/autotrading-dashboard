AI 트레이딩 팀을 만들었다. 15명의 AI 에이전트가 24시간 자동매매한다.

코딩 1도 모르는 사람도 따라할 수 있게 처음부터 끝까지 설명한다.



이게 뭔데?

AI 에이전트 15명이 팀을 이뤄서 코인 선물 자동매매를 한다.

사람으로 치면 이런 구조다:

애널리스트 9명 → "비트코인 지금 어떤 상태야?"
모더레이터 1명 → "9명 의견 종합해"
판사 1명 → "매매할지 말지 최종 결정"
리스크 매니저 → "너무 위험하면 거부권"
메모리 담당 → "과거에 비슷한 상황 어땠어?"
진화 담당 → "못하는 애 해고, 잘하는 애 승진"

이 팀이 1분마다 시장을 분석하고, 기회가 오면 자동으로 매매한다.



왜 만들었나?

사람이 차트 보면서 매매하면:
- 24시간 못 봄
- 감정에 휘둘림 (공포에 팔고, 탐욕에 삼)
- 같은 실수 반복

AI 팀은:
- 24시간 365일 쉬지 않음
- 감정 없음 (로직으로만 판단)
- 실수하면 자동으로 학습해서 다음엔 안 함



15명의 AI 에이전트

[분석팀 9명] 전부 무료로 돌아감

Trend — "지금 상승장이야 하락장이야?"
EMA, MACD로 추세 방향을 판단한다.

Momentum — "너무 많이 올랐어? 떨어졌어?"
RSI, 스토캐스틱으로 과매수/과매도를 감지한다.

Volatility — "변동성 폭발 직전이야?"
볼린저밴드, ATR로 변동성 상태를 분석한다.

Volume — "큰손들이 사고 있어? 팔고 있어?"
거래량, OBV로 자금 흐름을 추적한다.

Whale — "고래가 뭐 하고 있어?"
펀딩비, OI로 기관/고래 포지션을 감지한다.

Pattern — "이 차트 패턴 전에 봤는데..."
지지/저항, 차트 패턴을 인식한다.

Macro — "시장 분위기가 어때?"
뉴스 센티먼트, 공포지수를 분석한다.

CopyTrade — "잘하는 사람들은 뭐 하고 있어?"
상위 트레이더 포지션을 추적한다.

OnChain — "온체인 데이터가 뭐라고 해?"
MVRV, 거래소 입출금을 분석한다.

매매가 일어나는 과정

1분마다 이런 일이 벌어진다:

1) 시장 데이터 수집
BTC 15분봉, 1시간봉, 4시간봉 캔들을 가져온다.

2) 시장 국면 판단
"지금 강한 상승추세야" / "횡보장이야" / "급변장이야"
국면에 따라 전략이 바뀐다.

3) 신호 필터링 (무료)
RSI 과매도? 볼린저 하단 터치? 거래량 급증?
2개 이상 신호가 있어야 다음 단계로 넘어간다.

4) 분석팀 9명 동시 분석 (무료)
각자 BUY/SELL/HOLD + 확신도를 제출한다.
예: Trend=BUY(80%), Momentum=BUY(75%), Whale=SELL(60%)

5) 모더레이터 의견 종합 (무료)
"9명 중 7명이 BUY, 평균 확신도 72%"

6) 판사 최종 결정
과거 비슷한 상황 승률 확인 → "BUY, 확신도 78%"

7) 리스크 체크 (무료)
잔고 충분? 포지션 너무 많지 않아? → "승인"

8) 주문 실행
BTC/USDT LONG, 레버리지 25x, 마진 $20
손절: ATR 기반 자동 계산
익절: 손절의 2배 (손익비 2:1)



핵심 기능 6가지


1. ATR 기반 동적 손절/익절

"변동성에 따라 손절 거리가 자동으로 바뀐다"

변동성 낮을 때 (BTC가 잔잔할 때):
ATR = $300 → 손절 $450 거리, 익절 $900 거리

변동성 높을 때 (BTC가 출렁일 때):
ATR = $1,200 → 손절 $1,800 거리, 익절 $3,600 거리

왜 중요하냐면:
고정 손절은 잔잔한 시장에서 너무 넓고, 출렁이는 시장에서 너무 좁다.
ATR 기반이면 시장 상황에 맞게 자동 조절된다.


2. 수익권 SL 이동 (돈 지키기)

"수익이 나면 절대 손실로 안 돌아간다"

진입 직후:
→ SL = 진입가 - 1.5 ATR (일반 손절)

수수료 이상 수익 발생:
→ SL을 진입가로 이동 (이제 본전 보장)

1 ATR만큼 수익:
→ SL을 진입가 + 0.5 ATR로 이동 (최소 수익 확보)

2 ATR만큼 수익:
→ SL을 진입가 + 1 ATR로 이동 (수익 잠금)

수익 나다가 다시 마이너스로 돌아가는 일이 없어진다.
"아 그때 팔걸..." 하는 상황이 사라진다.


3. 멀티 타임프레임 정렬

"15분봉, 1시간봉, 4시간봉이 다 같은 방향이면 확신도 UP"

3개 전부 BUY → 확신도 +20% 부스트
2개 일치 → 확신도 +10%
다 다름 → 부스트 없음 (신호 약함)

큰 흐름과 작은 흐름이 같은 방향일 때만 적극적으로 들어간다.


4. 시장 국면 자동 전환

"추세장이냐 횡보장이냐에 따라 전략이 완전히 달라진다"

강한 추세장:
→ 포지션 x1.3 (공격적), 레버리지 x1.2, 트레일링 스탑으로 수익 극대화

일반 추세장:
→ 표준 포지션, 트레일링 스탑 사용

횡보장:
→ 포지션 x0.7 (보수적), 높은 확신도 필요, 고정 TP/SL

급변장:
→ 포지션 x0.5 (최소), 레버리지 x0.5, 거의 안 들어감

시장 상태를 자동으로 감지하고, 전략 파라미터가 자동으로 바뀐다.


5. 자기발전 루프 (제일 중요한 기능)

"매 거래마다 에이전트가 학습하고, 못하는 놈은 해고된다"

거래가 끝나면:

에이전트별 정답/오답 판정
→ "Trend가 BUY 했는데 실제로 올랐네 = 정답"
→ "Whale이 SELL 했는데 올랐네 = 오답"

가중치 업데이트
→ 정답 에이전트: 가중치 +5% (다음에 더 믿어줌)
→ 오답 에이전트: 가중치 -8% (다음에 덜 믿어줌)

누적 처벌
→ 5연속 오답: 수습 처리
→ 3주 연속 저성과: 해고

빈자리에 Recruiter가 새 에이전트를 영입한다.

시간이 지날수록 팀이 자동으로 강해진다.
잘하는 에이전트의 의견이 더 무겁게 반영되고,
못하는 에이전트는 자연스럽게 도태된다.


6. 펀딩비 역방향 전략

"다수가 한 방향으로 몰리면 반대로 간다"

펀딩비 > +0.01% (롱이 과열):
→ 모두가 롱 = 곧 내려갈 확률 높음 → SELL 신호 추가

펀딩비 < -0.01% (숏이 과열):
→ 모두가 숏 = 곧 올라갈 확률 높음 → BUY 신호 추가

군중의 반대편에 서는 전략이다.

세팅 방법 (처음부터)

준비물 4가지:
1) OKX 계정 + API 키
2) Anthropic API 키 (Claude)
3) 텔레그램 봇 토큰
4) Railway 계정 (무료 서버)


STEP 1: OKX API 키 발급

OKX 앱/웹 → 설정 → API → API 키 생성
권한: 거래, 읽기 (출금은 절대 X)
IP 제한: Railway 서버 IP 등록 (보안)
API Key, Secret Key, Passphrase 3개를 메모해둔다.


STEP 2: 환경변수 설정

거래소:
OKX_API_KEY=발급받은키
OKX_SECRET_KEY=시크릿키
OKX_PASSPHRASE=패스프레이즈
EXCHANGE_TESTNET=false

AI:
ANTHROPIC_API_KEY=sk-ant-...

텔레그램:
TELEGRAM_BOT_TOKEN=123456:ABC...
TELEGRAM_CHAT_ID=-100...

트레이딩:
TRADING_PAIRS=BTC/USDT:USDT,ETH/USDT:USDT
LEVERAGE_MIN=10
LEVERAGE_MAX=50
INITIAL_CAPITAL=100


STEP 3: Railway 배포

1) GitHub에 코드 올리기
2) Railway.app 가입 → New Project → Deploy from GitHub
3) 환경변수 전부 입력
4) Deploy 버튼 누르면 끝

Railway가 24시간 서버를 돌려준다.


STEP 4: 확인

텔레그램에 "자동매매 시스템 시작" 메시지가 오면 성공.
!status 명령어로 상태 확인하면 된다.
